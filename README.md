# site-m-guide

## 项目简介

`site-m-guide` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态 HTML 资源的集合与发布载体。  
每个页面保持独立、自包含，便于快速浏览、分享或离线使用。

## 目录说明

```
site-m-guide/
├── pages/          # 存放各独立 HTML 页面
├── assets/         # 页面共享的资源文件（如 CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件对应一个独立页面，命名清晰，互不依赖。
- `assets/`：存放多个页面可能共用的静态资源，避免重复。
- 根目录下的 `index.html` 可作为导航或说明页，方便直接通过 GitHub Pages 访问。

## 页面归档说明

所有归档的 HTML 页面均遵循以下原则：

- 每个页面为完整、独立的 HTML 文件，不依赖外部动态服务。
- 页面内部引用的资源优先使用相对路径，确保可在本地或静态托管环境中正常打开。
- 页面内容保持中立，不包含特定网站、品牌或营销信息。
- 归档目的仅为信息保存与结构展示，不承诺内容时效性或准确性。

## 维护说明

- 欢迎提交 Pull Request 添加新的独立页面或修正现有内容。
- 提交前请确保新页面不包含敏感信息、外部跟踪代码或非法内容。
- 资源文件请统一放入 `assets/` 目录，避免散落于页面文件夹中。
- 如需引用外部资源（如 CDN），请注明来源，并尽量使用可靠、长期稳定的链接。
- 本仓库维护者会定期检查页面可用性，但不对第三方资源链接的持续性负责。
